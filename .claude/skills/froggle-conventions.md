Code Organization Principles
1. Follow S.O.L.I.D. principles
2. Follow the YAGNI principle and keep things simple unless there's a requirement for it
3. Avoid duplication, but not at the cost of muddling intent
4. Remove dead code - Delete unused functions and imports
5. Keep naming semantic and simple
6. Make sure code is easy to read an understand
7. Try to retain a single source of truth

Coding Style References:
1. 99 Bottles of OOP by Sandi Metz
2. Object Oriented Design by Sandi Metz
3. Refactoring by Martin Fowler
